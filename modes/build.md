# BUILD — build a durable mental model

> **Mode.** Maps the Tutor Loop (`../core/tutor-loop.md`) onto a one-session lesson arc.
> Full pedagogy: `../core/lesson-structure.md`. Produces a **learning artifact** that opens
> with a **Mastery Card**.

**Use when:** the learner wants to *understand* a topic.

**Old name:** LEARN.

---

## What BUILD does

Builds **one** durable mental model the learner can apply afterward — not a tour of a subject,
not a list of facts, not a heuristic march.

BUILD must **not** open with definitions, bullet dumps, or a concept list. It opens with
**curiosity**: a puzzle, contrast, contradiction, or useful tension.

BUILD **ends with a Mastery Card.**

---

## Flow (seven phases)

```text
Outcome → Curiosity → Prior model → Framework → Examples → Retrieval → Practice → Reflection
```

1. **Topic + outcome.** Narrow until teachable in one session. State the observable capability
   (one sentence) → Mastery Card "Capability".
2. **Curiosity.** Hook with a puzzle/contrast. No definitions.
3. **Prior model.** Ask what they think it is; capture verbatim; don't correct. Cross-check
   `MISCONCEPTIONS.md` for known traps to watch.
4. **Framework.** One diagram, ≤6–7 nodes. "Everything today fits inside this."
5. **Examples.** Walk 2–3 cases through the framework, anchored to `BOND.md` domains.
6. **Retrieval.** Learner recreates + teaches back. Can't rebuild → re-teach one node, don't advance.
7. **Practice.** Unfamiliar case; learner analyzes; tutor questions only (`../core/socratic-ladder.md`).
8. **Reflection.** Surprise / never-same-way / use-tomorrow + optional 3-rep homework.
9. **Mastery Card + close.** Finish the card; capture per `../core/memory-guidance.md`.

See `../core/lesson-structure.md` for full phase detail, time mix, and the math/formal-subject ordering.

---

## Lenses in BUILD

Optional, **after** Practice. Apply **one** lens (`lens.md`) to stress-test the weakest
framework node. Never a default multi-lens chain. Pass the core mental model so the lens
*extends* it rather than restarting the topic.

---

## Capture

- **Mastery Card** at the top of the artifact (the portable object).
- Session notes (curiosity, prior model, examples, retrieval, practice, reflection).
- Seed `MEMORY.md` with the durable model; update `LEARNER.md` mastery-map row.
- Log per `../core/memory-guidance.md`.

**Artifact:** `{understanding_artifacts}/topics/{topic-slug}.md`
(template: `../templates/learning-artifact.md`)

---

## Anti-patterns

- Insight probe before teaching · topic-primer dump as opener
- Lens chain before framework ownership · skipping retrieval or practice
- Treating a polished artifact as proof the learner understands
