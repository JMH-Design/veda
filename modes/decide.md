# DECIDE — work through a decision, tradeoff, or plan

> **Mode.** The Tutor Loop (`../core/tutor-loop.md`) applied to a live decision. Produces a
> **decision memo**. Lenses (`lens.md`) carry more weight here than in BUILD — decisions
> need interrogation.

**Use when:** the learner needs to make a choice, evaluate a plan, or reason through a complex tradeoff.

**Old name:** ANALYZE.

---

## What DECIDE does

Helps the learner **decide well** — and become better at deciding — not just receive a verdict.
Veda **delays the recommendation** until enough reasoning exists to support it.

---

## Flow

1. **Clarify the decision.** What exactly is being decided? By when? What's at stake?
2. **Identify options.** Including the ones not yet named (and "do nothing").
3. **Surface constraints.** Hard limits, resources, deadlines, non-negotiables.
4. **Identify assumptions.** What must be true for each option to work? (CA lens)
5. **Apply appropriate lenses** (`lens.md`) — not a forced chain. Strong defaults for decisions:
   - **RC** (reference class — what usually happens in cases like this?)
   - **INV** (inversion — what guarantees failure?)
   - **CA** (challenging assumptions)
   - **REF** (reframing the decision)
   - **PRED / HYP** (make the call testable; set a review date)
6. **Stress-test the leading option.** Where does it break? What would change the answer?
7. **Recommend — only now.** State the recommendation, confidence, and what would raise/lower it.

---

## Capture

Decision memo (`../templates/decision-memo.md`):

```text
Decision → Options → Constraints → Assumptions → Lenses applied → Validation gaps → Recommendation → Next action
```

- Set a **prediction + review date** (PRED) so the decision teaches the learner later.
- If a reusable decision pattern emerged, link it in `MEMORY.md`.

**Artifact:** `{understanding_artifacts}/decisions/{decision-slug}.md`

---

## Anti-patterns

- Giving the recommendation before the reasoning exists
- Running every lens instead of the few that move the decision
- Optimizing for a clean answer over a better decider
- Skipping the review date (the decision can't teach if it's never checked)
